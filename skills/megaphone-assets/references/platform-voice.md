## Bluesky

- **Char limit:** 300
- **Tone:** developer-friendly, conversational, slightly self-deprecating
- **No hashtags by convention**
- **Shape:** hook in line 1, one concrete detail, link last

## X (Twitter)

- **Char limit:** 280 per tweet, threads encouraged
- **Tone:** punchier than Bluesky
- **Threads:** payoff-promising hook in tweet 1, link in LAST tweet
- **Hashtags:** 0–2, only if extremely relevant
- **API note:** ~$0.01/post since Feb 2026, bot accounts must disclose

## LinkedIn

- **Char limit:** 3000, sweet spot 1200–2000
- **Tone:** polished, story-led, not announcement-led
- **Format:** short paragraphs (1–3 sentences), generous whitespace
- **End with a question** — drives comment engagement
- **Hashtags:** 3–5 at the bottom

## dev.to

- **Length:** 800+ words, sweet spot 1500–2500
- **Title:** promises a payoff (a number, a takeaway, a confession)
- **Format:** H2/H3 sections, code snippets, intro/body/conclusion
- **Tags:** 4 max — stack tags + category tags (`#showdev`, `#opensource`)
- **Cover image:** strongly recommended

## Threads (Meta)

- **Char limit:** 500
- **Tone:** casual, visual-first
- **Best for:** consumer-facing products

## Mastodon

- **Char limit:** 500 default
- **Tone:** friendly, hashtag-positive
- **Hashtags:** 2–4 drive federated discovery

## Universal rules

1. Match the user's voice samples from `.megaphone/profile.json`
2. One concrete detail (number, name, file) per piece
3. Active verbs over passive
4. One exclamation point max, only when something genuinely shipped
5. No emoji unless voice samples use them
6. No "I'm excited to announce" — ever
7. No fabricated outcomes
8. The opening line is the whole game on short-form
